Agent Wrapper 包图

模块组织、命名空间与分层架构
Go module github.com/smallnest/agent-wrapper。根包定义公开接口，子包按职责分层：类型层 → 支撑层 → Provider 实现层。
LAYER ARCHITECTURE

分层结构

  • Layer 1: 公开 API — agent-wrapper + types + cmd
  • Layer 2: 支撑层 — harness (审批/预算/压缩) + process (子进程)
  • Layer 3: Provider 实现 — 8 个独立包，互不依赖
  • 依赖方向: Layer 3 → Layer 2 → Layer 1 (单向)
PACKAGE DEPENDENCIES

包间依赖

  • 所有包 → types (类型定义)
  • root + provider/* → process (子进程管理)
  • root → harness (审批/预算)
  • provider/* → root (Agent 接口)
  • cmd → root (公开API) + provider/* (注册)
