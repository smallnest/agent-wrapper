组件间消息协作关系与编号顺序
以协作网络视角展示组件间的消息传递，编号标明调用顺序。强调组件间的链接关系而非严格时序。
COLLABORATION

协作关系

  • Caller → Orchestrator: 发起调用，消费事件
  • Orchestrator → Agent: 委托执行，审批介入
  • Agent → AgentProcess: 管理子进程生命周期
  • AgentProcess → Agent CLI: stdin/stdout 通信
MESSAGE FLOW

消息顺序

  • 1-2: 调用链下行（caller → agent）
  • 3-5: 子进程启动（创建 → 运行）
  • 6: NDJSON 流上行（CLI → scanner）
  • 7: 事件转发（orchestrator → caller）
  • 8: 审批拒绝（orchestrator → caller DENIED）
