Agent Wrapper 定时图

AgentProcess 与 Orchestrator 状态随时间变化的时间约束
展示一次 Agent 调用的 4 条生命线随时间展开的状态变化。横轴为时间线（ms），标注关键事件的持续时间和超时约束。
TIMING LIFELINES

时间线

  • AgentProcess: Created → Running → Exited OK / Exit Error
  • Orchestrator: Idle → Processing → Completed (or compress+retry self-loop)
  • ApprovalHandler: Inactive → Active(allow/deny) → Inactive (per tool call)
  • Agent CLI stdout: system/init → text_delta → tool_use → result → close
TIME CONSTRAINTS

时间约束

  • 子进程启动: ~100ms (resolveBinary + cmd.Start)
  • 典型 turn: 2–10s (取决于 LLM 响应速度)
  • SIGTERM → SIGKILL 超时: 5s (硬编码)
  • 上下文溢出重试: 最多 3 次
  • 事件 channel 缓冲: 64 (非阻塞发送)
