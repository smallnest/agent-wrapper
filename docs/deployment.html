Agent Wrapper 部署图

运行时环境、进程拓扑与通信协议
agent-wrapper 作为 Go 库嵌入宿主进程，通过子进程驱动各 Agent CLI。无外部中间件依赖，所有状态在内存中管理。
DEPLOYMENT TOPOLOGY

节点与进程

  • Go Runtime — 单一宿主机进程（开发者笔记本或 CI Runner）
  • Agent CLI 子进程 × 8 — 每个 Provider 对应一个外部 CLI 二进制
  • 无中间件依赖 — Registry、Session 均在内存中
  • 所有状态进程内管理，无 Redis/DB
COMMUNICATION

协议与通道

  • Go ↔ CLI: stdin/stdout pipe (文本协议)
  • NDJSON / JSON / Plain text 取决于 Provider
  • CLI ↔ LLM API: HTTPS (各 CLI 自行管理)
  • CLI 安装: npm / cargo / pip (一次性)
